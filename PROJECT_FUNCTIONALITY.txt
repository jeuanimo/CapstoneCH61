1. PROJECT OVERVIEW
================================================================================

This is a comprehensive fraternity chapter management and e-commerce platform
built with Django. The application serves as a complete digital hub for chapter
operations, member management, financial tracking, communications, and 
merchandise sales.

TARGET USERS:
- Public visitors (prospective members, community)
- Active chapter members
- Chapter officers (administrative privileges)
- Treasurer (financial management)
- Chapter administrators

CORE FUNCTIONALITY AREAS:
- Public website with chapter information
- Member portal with social features
- Officer management console
- Financial/dues management with Stripe integration
- SMS notifications via Twilio
- Email communication system
- E-commerce boutique shop
- AI-powered public chatbot

10. SECURITY FEATURES
================================================================================

AUTHENTICATION & AUTHORIZATION
- Django built-in authentication
- Password hashing (PBKDF2)
- Session management
- CSRF protection
- Permission decorators:
  * @login_required
  * @user_passes_test(is_officer_or_staff)
  * Staff-only access

ACCESS CONTROL LEVELS
- Public (no auth)
- Member (authenticated users)
- Officer (is_officer=True or is_staff=True)
- Treasurer (financial permissions)
- Admin (full access)

DATA PROTECTION
- Input validation on all forms
- SQL injection prevention via Django ORM
- XSS protection via template auto-escaping
- Secure password reset
- File upload validation
- Safe redirect URL validation

AUDIT TRAILS
- User action logging
- Login attempt tracking
- Payment transaction logs
- SMS delivery logs
- Document access tracking

OWASP TOP 10 COMPLIANCE
- Broken Access Control - Permission checks on all views
- Cryptographic Failures - Hashed passwords, HTTPS ready
- Injection - Parameterized queries via ORM
- Insecure Design - Security-first architecture
- Security Misconfiguration - Production settings guide
- Vulnerable Components - Dependency tracking
- Authentication Failures - Strong password requirements
- Data Integrity - Validated inputs and outputs
- Logging Failures - Comprehensive logging
- SSRF - URL validation for redirects

GDPR COOKIE CONSENT
- Cookie consent banner on first visit
- Three consent tiers:
  * Necessary Only - Essential cookies only
  * Analytics - Includes page view tracking
  * All Cookies - Full functionality
- Consent stored in database (CookieConsent model)
- Middleware-based consent checking
- Re-prompt option for preference changes

ROBOTS.TXT (Web Crawler Control)
- Allows public pages:
  * /, /about/, /contact/, /events/
  * /chapter-programs/, /programs/
  * /chapter-leadership/, /shop/
- Blocks private areas:
  * /portal/, /admin/, /login/, /logout/
  * /signup/, /activate/, /chatbot/
  * /media/member_photos/, /media/documents/
- 10-second crawl delay configured
