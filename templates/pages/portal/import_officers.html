CSV Format Guide

Your CSV file should have a header row with the following columns:

Required Fields:
  • full_name - Officer's full name
  • position - Position title (e.g., President, Vice President, Secretary, Treasurer, etc.)
Optional Fields:
  • email - Officer's email address
  • phone - Phone number
  • bio - Brief biography or description
  • position_custom - Custom position title (if position doesn't match standard titles)
  • display_order - Numeric order for display (lower numbers first, default: 0)
Supported Position Titles:

President, Vice President, Secretary, Treasurer, Parliamentarian, Chaplain, Historian, Sergeant at Arms, Board Member

Note: If a position doesn't match the above, it will be saved as "Other Position"

Sample CSV Format:
full_name position email phone display_order
John Smith President john.smith@example.com 555-0100 1
Jane Doe Vice President jane.doe@example.com 555-0101 2
Bob Johnson Treasurer bob.j@example.com 555-0102 3
Important Notes:
  • All officers imported will be marked as active
  • If an officer with the same name and position already exists (and is active), they will be skipped
  • The system supports both uppercase and lowercase column names
  • Empty optional fields will be left blank in the system
